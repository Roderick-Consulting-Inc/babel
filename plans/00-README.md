# Babel — Planning Documents

Babel is a programmatic generator of esoteric programming languages: a methodology, a parameter schema, and a runtime that emits (a) runnable interpreters, (b) transpilers to a chosen base, and (c) specification pages.

These planning documents precede any code. Their purpose is to fix the framing — what we are building, why, where it sits in prior art, and what value it brings — before architectural or implementation decisions force us to commit.

## Two artifacts, one research project

This planning folder describes two distinct artifacts that share a single intellectual project:

1. **Babel** — the language generator (this repo).
2. **Inflexión** — a hand-crafted Spanish-grammar-as-semantics esoteric language ([Inflexion repository](https://github.com/Roderick-Consulting-Inc/inflexion); the directory name is the ASCII spelling). It is intentionally **not** generated by Babel; we want the language design free of any tooling limits we have not yet discovered.

The two artifacts share the foundation documents (`01`–`03`) below. Each gets its own white paper (`04`, `05`).

## Citation infrastructure

White papers (`04`, `05`) use **Chicago author-date** as their citation style. The shared bibliography lives at `references.bib` and is BibTeX. White-paper drafts are written in markdown with `[@key]` citation syntax and rendered through pandoc:

```bash
pandoc 04-whitepaper-babel.md \
  --citeproc \
  --bibliography=../references.bib \
  --csl=../csl/chicago-author-date.csl \
  -o 04-whitepaper-babel.pdf
```

The CSL file (`csl/chicago-author-date.csl`) is fetched from the [Citation Style Language repository](https://github.com/citation-style-language/styles) when the first paper is rendered; pandoc's bundled copy is also acceptable.

Planning documents (`01`–`03`) keep things light — inline footnoted URLs only, no full citations. Sources we know we will cite later are seeded into `references.bib` as we encounter them, so 04 and 05 can pull from a populated bibliography without re-collecting.

## Conventions in these docs

- The Spanish language is referred to as `Inflexión` until named.
- "Esolang" without qualification means *esoteric programming language* in the sense of esolangs.org.
- "Romance grammar" means the family of grammatical features Spanish inherits from Latin: gendered nouns, verb conjugation by person/number/tense/mood/aspect, clitic pronoun systems, ser/estar copular split, diminutive/augmentative morphology, voseo and other dialect-specific forms. We will be specific in `05` about which features carry semantic weight.
- `[TBD]` marks open questions for the human author to resolve before the doc is considered drafted.
